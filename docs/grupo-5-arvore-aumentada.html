01 · fundamentos

Antes da árvore, o que é uma estrutura de dados?

Uma estrutura de dados é uma forma organizada de representar valores e as operações que podemos realizar sobre eles. A mesma informação pode ser armazenada de formas diferentes, com custos diferentes.

Regra de linguagem deste guiaQuando aparecer a palavra chave, imagine um identificador numérico de livro. A árvore guarda e compara esse ID. Ela não guarda o livro completo.

Dicionário de bolso

Use estas traduções sempre que um termo técnico aparecer. O código mantém os nomes originais porque eles são a interface da implementação.

rank(k)Quantos IDs de livros são menores que k. Exemplo, em [10, 20, 30], rank(30) = 2.
select(i)Qual ID ocupa a posição i na ordem. O índice usado aqui começa em zero.
range_agg(a, b)Um resumo dos IDs entre a e b. No grupo 5, o resumo é o maior ID do intervalo.
subárvoreUm nó junto com todos os nós abaixo dele, como uma pequena árvore dentro da árvore.
alturaQuantos níveis existem no caminho mais longo a partir de um nó.
rotaçãoUma reorganização local dos nós para manter a árvore equilibrada.
quente / frioQuente significa escolhido muitas vezes. Frio significa escolhido poucas vezes.
hit / missHit encontra o ID. Miss não encontra o ID no estado atual.

Contrato antes da implementação

O contrato diz que uma pilha oferece push e pop. Ele não diz se a pilha usa uma lista Python, um arranjo fixo ou outra representação. Essa separação permite comparar alternativas sem mudar quem usa a estrutura.

A mesma ementa em uma frase

Arranjos dão acesso direto. Listas conectam elementos. Pilhas seguem LIFO. Filas seguem FIFO. Mapas e dicionários ligam chaves a valores. Árvores representam hierarquias e ordenação. Classificação, pesquisa e recursão são formas de processar esses dados.

Vocabulário mínimo

Chave é o nome técnico do identificador numérico usado para identificar ou ordenar um elemento. Neste projeto, pense em chave como ID de livro. Nó é uma unidade de uma estrutura ligada. Subárvore é um nó junto com todos os seus descendentes. Complexidade descreve como o custo cresce quando o tamanho da entrada cresce.

Arranjos e listas

Índice ou sequência

values[2] acessa diretamente uma posição. append adiciona ao fim. Inserir no meio pode deslocar vários elementos.

Pilhas e filas

Ordem de atendimento

Uma pilha usa LIFO, como append + pop. Uma fila usa FIFO, preferencialmente deque + popleft.

Mapas e classificação

Chave, valor e ordem

dict resolve chaves exatas. sorted(values) cria uma sequência ordenada. Uma árvore será útil quando ordem e atualização precisarem coexistir.

Pesquisa em Python

def linear_search(values, key):
    """Retorna o índice de key ou -1 se não encontrar."""
    for index, value in enumerate(values):
        if value == key:
            return index
    return -1

A pesquisa linear examina itens até encontrar a chave. Em uma sequência ordenada, a busca binária reduz o intervalo pela metade. Em uma BST, a propriedade de ordenação guia o caminho.

Classificação é uma transformação

records = [("Ana", 8), ("Bia", 10), ("Caio", 7)]
ordered = sorted(records, key=lambda record: record[1])

A classificação facilita pesquisa e análise, mas tem custo. No trabalho, a ordem ordenada é usada para provocar o caso patológico de uma BST ingênua antes de comparar com uma estrutura balanceada.

A ponte para árvores

Recursão é pensar no mesmo problema em menor escala.

Uma árvore é naturalmente recursiva: cada filho pode ser visto como a raiz de uma árvore menor. Toda função recursiva precisa de um caso-base e de um passo que aproxima a execução desse caso.

buscar(nó, k) = buscar(nó.esquerda, k) ou buscar(nó.direita, k)

Na BST, a propriedade de ordenação elimina um dos caminhos em cada decisão. Por isso, a busca visita um caminho, não todos os nós, quando a árvore está balanceada.

Modelo mental da busca

  1. Se o nó é vazio, a chave não está presente.
  2. Se a chave é igual ao nó, encontramos.
  3. Se a chave é menor, descemos à esquerda.
  4. Se a chave é maior, descemos à direita.

O mesmo padrão aparece em insert, delete, rank, select e range_agg. 

def search(node, key):
    """Retorna True se key estiver na subárvore enraizada em node."""
    if node is None:                 # caso-base
        return False
    if key == node.key:
        return True
    if key < node.key:
        return search(node.left, key)  # problema menor
    return search(node.right, key)     # problema menor
02 · a estrutura obrigatória

BST + balanceamento + metadados

O projeto pede uma árvore binária de busca balanceada aumentada. São três ideias empilhadas. Cada uma precisa continuar correta para a próxima funcionar.

BST

Ordenação local

Na subárvore esquerda ficam chaves menores. Na direita, chaves maiores. Essa regra transforma uma busca em decisões sucessivas.

Balanceamento

Altura sob controle

AVL, rubro-negra ou treap. A escolha é do grupo. O objetivo é evitar que a árvore vire uma lista longa.

Aumento

Resumo na subárvore

Cada nó guarda size e aggregate. No grupo 5, o agregado é o maior valor da subárvore.

O estado de um nó aumentado

class Node:
    def __init__(self, key):
        self.key = key
        self.left = None
        self.right = None
        self.height = 1
        self.size = 1
        self.subtree_max = key

O ponto decisivo é que height, size e subtree_max não são dados independentes. Eles são resumos calculados a partir dos filhos.

size(n) = 1 + size(n.left) + size(n.right)
max(n) = max(n.key, max(n.left), max(n.right))
balance(n) = height(n.left) − height(n.right)

Rotação não é mágica

Uma rotação muda poucas referências para restaurar a altura. Depois dela, o nó que desceu deve ser atualizado antes do nó que subiu, porque o resumo do segundo depende do primeiro.

!
Falha silenciosaSe a rotação esquecer subtree_max, a busca ainda pode funcionar, mas range_agg passa a responder errado.

Atualização local em Python

def update(node):
    """Recalcula todos os resumos de node."""
    left = node.left
    right = node.right
    node.height = 1 + max(height(left), height(right))
    node.size = 1 + size(left) + size(right)
    node.subtree_max = max(node.key, subtree_max(left), subtree_max(right))

Como ler o laboratório

Após uma inserção, observe a mensagem de rotação. Em uma AVL, o fator de balanceamento fica entre −1 e +1. A árvore pode mudar de forma, mas sua ordenação e seus resumos devem permanecer válidos.

A demo usa números pequenos para tornar o mecanismo visível. Ela não representa a execução de centenas de milhões de chaves.

Versão Python da operação de agregado

def range_max(node, lower, upper):
    """Retorna o maior valor de [lower, upper]."""
    if node is None or node.subtree_max < lower:
        return None
    if node.key < lower:
        return range_max(node.right, lower, upper)
    if node.key > upper:
        return range_max(node.left, lower, upper)
    values = [node.key,
              range_max(node.left, lower, upper),
              range_max(node.right, lower, upper)]
    values = [value for value in values if value is not None]
    return max(values)

Esta versão usa subtree_max para podar subárvores que não podem conter uma resposta. Uma implementação mais agressiva também pode retornar o resumo de uma subárvore inteira quando os limites provarem que ela está totalmente dentro do intervalo.

O núcleo obrigatório

Seis operações, seis perguntas diferentes.

insert(k)

Insere k mantendo a ordenação, o balanceamento e todos os metadados.

delete(k)

Remove k. Se o nó tem dois filhos, substitui pelo sucessor/predecessor e atualiza o caminho.

search(k)

Informa se k está presente seguindo a propriedade da BST.

rank(k)

Conta quantas chaves são menores que k usando size para saltar subárvores.

select(i)

Encontra a i-ésima menor chave, normalmente com índice 0-based.

range_agg(a, b)

Para o grupo 5, retorna o máximo entre as chaves do intervalo fechado [a, b].

Complexidade esperada de uma AVL

Operação Limite teórico Por que
search, insert, delete O(log n) Visitam um caminho de altura limitada; inserção/remoção podem fazer rotações.
rank, select O(log n) O tamanho das subárvores permite descartar blocos inteiros.
range_agg simples O(log n + k) Visita a fronteira e os k valores relevantes; subtree_max permite podar partes que não podem responder.
Atualizar um nó O(1) Combina apenas os resumos dos dois filhos.

Os limites descrevem crescimento assintótico. Não predizem sozinhos o tempo em nanosegundos.

04 · carga de trabalho

De chaves reais a um trace reproduzível.

O SOSD fornece dados reais em formato binário e somente para leitura. O script local transforma essas chaves em inserções, remoções e buscas. O trace e o oráculo tornam a corretude verificável.

Os quatro conjuntos reais

Nome no trabalho Arquivo SOSD Característica
amzn books_200M_uint32 Popularidade de vendas de livros; distribuição moderadamente enviesada.
face fb_200M_uint64 Identificadores de usuários; lacunas grandes e regulares.
wiki wiki_ts_200M_uint64 Timestamps; quase denso, com agrupamentos temporais.
osm osm_cellids_800M_uint64 Lacunas muito irregulares; o mais difícil.

Formato e escala

O arquivo binário começa com 8 bytes contendo a quantidade de chaves em uint64 little-endian. Depois vêm as chaves em uint32 ou uint64.

Os conjuntos reais têm de 200 a 800 milhões de inteiros. O arquivo osm chega a cerca de 6,4 GB, por isso --carga-maxima lê apenas as primeiras N chaves sem carregar tudo na memória.

Importante: SOSD originalmente é uma carga somente de leitura. Inserções e remoções são decisões do gerador do trabalho.

Por que a verificação é transitiva?

O resultado de uma busca depende do estado produzido pelas operações anteriores. Se uma remoção falha, a chave removida pode reaparecer como FOUND em uma busca posterior.

O enunciado define a semente como o número do grupo. Para esta instância, use sempre --semente 5 nas comparações oficiais.

O que o grupo precisa conseguir explicar

  • Por que a ordem ordenada é patológica para uma BST ingênua e como a AVL reage.
  • Como a concentração de acessos em theta = 0.99 altera a carga, a localidade e a leitura dos percentis.
  • Por que subtree_max responde ao agregado do intervalo sem varrer tudo em todos os casos.
  • Como a mistura 45:25:30 muda o número de inserções, remoções e buscas observadas.
  • Por que o resultado medido na máquina do grupo é necessário para o relatório.
Atenção ao recorte: não troque silenciosamente o conjunto, a seed, o theta, o mix, o agregado ou a ordem de inserção. Se fizer uma execução exploratória diferente, identifique-a como tal.

06 · medir e interpretar

Teoria é uma expectativa. Medição é evidência.

O estudo empírico exigido compara escalas, enviesamento, ordem de inserção, uma linha de base e os limites teóricos. O gráfico abaixo é deliberadamente conceitual. Seus números reais precisam vir da máquina do grupo.

Crescimento esperado

A AVL deve se aproximar de O(log n) para as operações de caminho. Uma lista ordenada como baseline evidencia o custo linear.

O que registrar em cada execução

  • n, número de operações, proporção real I/D/S e seed.
  • Tempo por operação, com média, p50 e p99.
  • Máquina, sistema, versão do Python/compilador e configuração.
  • Memória, duração total, warm-up e repetições.
  • Erros do oráculo e como foram investigados.

Comparações obrigatórias

  1. Quatro ordens de grandeza de n, até o limite suportado.

Regras que protegem a avaliação

  • Ferramentas de auxílio são permitidas para o código, mas o grupo deve compreender todo o conteúdo entregue.

  • Identifique qualquer parte cuja autoria não seja do grupo.
  • Forneça o passo a passo para outra pessoa reproduzir a solução.
