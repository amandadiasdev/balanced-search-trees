#### 1. Por que `insert(k)` desce até uma posição vazia?

Em uma árvore de busca, uma nova chave sempre entra como uma folha.

Isso não significa que a chave precise ser “extrema”. Ela pode ser intermediária na ordem, mas ainda assim ocupará uma nova posição vazia entre dois nós existentes.

Exemplo, suponha:

```text
    20
      \
       40
```

Agora queremos inserir `30`.

`30` é intermediário entre `20` e `40`, mas não substitui nenhum desses nós:

```text
    20
      \
       40
      /
     30
```

O caminho foi:

1. `30 > 20`, então descemos pela direita.
2. `30 < 40`, então descemos pela esquerda.
3. A esquerda de `40` está vazia.
4. Criamos `30` ali.

A posição vazia representa o local correto para a chave segundo a regra da árvore.

A recursão apenas expressa esse processo:

```python
insert(node, key):
    if node is vazio:
        criar novo nó com key
    elif key < node.key:
        node.left = insert(node.left, key)
    elif key > node.key:
        node.right = insert(node.right, key)
```

Depois que o novo nó é criado, a função retorna pelo caminho inverso, atualizando `height`, `size`, `subtree_max` e, em uma AVL, realizando rotações se necessário.

Portanto, “intermediário” significa intermediário na ordem dos valores. Não significa que devemos inserir a chave dentro de um nó existente. A árvore não desloca os nós como um vetor. Ela cria uma nova ligação em uma posição vazia.

#### 2. E se o sucessor na remoção tiver filhos?

O sucessor é a menor chave da subárvore direita.

Ele nunca pode ter um filho esquerdo. Se tivesse, esse filho seria ainda menor e seria o verdadeiro sucessor.

Mas ele pode ter um filho direito.

Exemplo:

```text
      40
        \
         60
        /
       50
         \
          55
```

Queremos remover `40`.

O sucessor é `50`. Primeiro copiamos `50` para o lugar de `40`:

```text
      50
        \
         60
        /
       50
         \
          55
```

Agora precisamos remover a ocorrência antiga de `50`. Como ela tem um filho direito, `55` ocupa seu lugar:

```text
      50
        \
         60
        /
       55
```

O resultado final não possui duas cópias de `50`.

O procedimento é:

1. Encontrar o sucessor.
2. Copiar apenas a chave do sucessor para o nó que será removido.
3. Remover o sucessor da posição original.
4. Se o sucessor tiver filho direito, conectar esse filho ao pai do sucessor.
5. Atualizar metadados e rebalancear o caminho de volta.

O caso é mais simples porque o sucessor nunca tem dois filhos. Ele possui zero ou um filho, sempre o direito.

#### 3. Como funciona `select(i)`?

`select(i)` procura a chave que ocupa a posição `i` na ordem crescente.

O índice começa em zero:

```text
chaves ordenadas: [20, 30, 40, 50, 70]

select(0) = 20
select(1) = 30
select(2) = 40
select(3) = 50
select(4) = 70
```

Considere esta árvore:

```text
      50
     /  \
   30    70
  /  \
20    40
```

A ordem crescente é:

```text
20, 30, 40, 50, 70
```

No nó `50`, existem três chaves na subárvore esquerda:

```text
20, 30, 40
```

Logo, `50` ocupa a posição `3`.

A regra é:

```python
left_size = size(node.left)
```

- Se `i == left_size`, o nó atual é a resposta.
- Se `i < left_size`, procuramos na esquerda.
- Se `i > left_size`, procuramos na direita com:

```python
i = i - left_size - 1
```

Por exemplo, para `select(2)`:

1. No nó `50`, `left_size = 3`.
2. Como `2 < 3`, vamos para a esquerda.
3. No nó `30`, `left_size = 1`.
4. Como `2 > 1`, vamos para a direita.
5. Ajustamos o índice:

```text
2 - 1 - 1 = 0
```

6. Chegamos ao nó `40`, cuja esquerda tem tamanho zero.
7. Como o índice restante é zero, a resposta é `40`.

O `select` não usa um vetor ordenado separado. Ele usa o campo `size` armazenado em cada nó para saltar subárvores inteiras.

Uma forma curta de lembrar:

- `rank(k)` pergunta: “quantas chaves vêm antes de `k`?”
- `select(i)` pergunta: “qual chave está na posição `i`?”

Eles são operações inversas sobre a ordem das chaves.

### Codex

Não. `search(k)` e `select(i)` recebem tipos de entrada diferentes:

- `search(k)` recebe uma chave e pergunta se ela existe.
- `select(i)` recebe uma posição e retorna qual chave ocupa essa posição na ordem crescente.

Exemplo:

```text
        50
       /  \
     30    70
    /  \
   20   40
```

A ordem crescente é:

```text
[20, 30, 40, 50, 70]
```

Então:

```text
search(2)  → NOT_FOUND
select(2)  → 40
```

O `2` de `search(2)` é interpretado como uma chave. O `2` de `select(2)` é interpretado como um índice.

Para implementar `select`, precisamos usar `size`:

1. Conte quantos nós existem na subárvore esquerda.
2. Se `i` for igual a essa quantidade, o nó atual é a resposta.
3. Se `i` for menor, continue pela esquerda.
4. Se `i` for maior, vá para a direita descontando:

```text
size(esquerda) + 1
```

No exemplo, em `50`, existem três chaves à esquerda. Como `select(2)` procura uma posição menor que 3, descemos para `30`. Em `30`, existe uma chave à esquerda, `20`. Como a posição 2 ultrapassa esse bloco, vamos para `40` com índice restante:

```text
2 - 1 - 1 = 0
```

Logo, `select(2) = 40`.

Uma relação útil é:

```text
rank(40) = 2
select(2) = 40
```

Ou seja, `rank` transforma uma chave em posição, enquanto `select` transforma uma posição em chave.
